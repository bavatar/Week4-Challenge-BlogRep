Java Encapsulation

28 July 2017 
Encapsulation. This is the practice of keeping fields within a class private,
then providing access to them via public methods. It’s a protective barrier
that keeps the data and code safe within the class itself. This way, we can
re-use objects like code components or variables without allowing open access
to the data system-wide.
How Encapsulation Works
 
Encapsulation lets us re-use functionality without jeopardizing security.
It’s a powerful OOP concept in Java because it helps us save a lot of time.
For example, we may create a piece of code that calls specific data from a
database. It may be useful to reuse that code with other databases or
processes. Encapsulation lets us do that while keeping our original data private
